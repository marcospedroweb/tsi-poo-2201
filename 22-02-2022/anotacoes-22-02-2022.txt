Diagramas (estaticos)
  Diagrama de componentes
    - Descreve como os componentes são conectador para formar componentes maiores
    - Demonstra onde conectar/conversa com outro componente (Conector/Plug)
    - Usado para delimitar componentes de outros

  Diagrama de Estrutura Composta
    - Serve para conseguir especificar alguma caracteristica relacionada com o algoritmo em uma determinada classe 
    
  Diagrama de implantação
    - Tudo que é necessario para aplicar a nossa aplicação
  
  Diagrama de objeto
    - Semelhante ao diagrama de classes, o que muda, é que ele mostra o nome da classe e qual objeto foi criado a partir dessa classe

  Diagrama de pacote (dependencias)
    - Mostra tudo que necessario para rodar a aplicação

  Diagrama de perfil (metamodelo)

Diagramas (dinâmicos)
  Descrever caracteristicas da nossa aplicação
  
  Diagrama de atividade
    - Fluxo com atividades paralelas (Representado por uma barra grossa)
  
  Diagrama de Sequência
    - Representa uma sequência de processos e as mensagens trocadas no tempo
    - Tem como função representar a intereção dos objetos

  Diagrama de Caso de Usado
    - Representa a intereção de um usuario com o sistema
    - Mostra a varias maneiras que podemos interagir com o sistema

  Diagrama de comunicação
    - Simplificação da primeira versão do OML

  Diagrama de Visão Geral da Interação
    - Parecido com o diagrama de atividades, mas com esse é possivel fazer uma explicação mais detalhada de uma parte especificar

  Diagrama de Estado (State Machine)
    - Voltado para o hardware
    - Mostra o estado da maquina ou aparelho
  
  Diagrama de tempo
    - Detalha o comportamento dos objetos durante um determinado periodo
